# Framework Examples

This directory contains clean, focused examples demonstrating pydantic-forms with different web frameworks.

## Quick Start

### Flask Example
```bash
# Run Flask example
cd examples
python flask_example.py

# Visit:
# http://localhost:5000/ - Home page
# http://localhost:5000/bootstrap/login - Simple login form
# http://localhost:5000/bootstrap/pets - Bootstrap pets demo
# http://localhost:5000/material/pets - Material Design pets demo
```

### FastAPI Example
```bash
# Install FastAPI dependencies first
pip install fastapi uvicorn python-multipart jinja2

# Run FastAPI example
cd examples
python fastapi_example.py

# Visit:
# http://localhost:8000/ - Home page
# http://localhost:8000/bootstrap/login - Simple login form
# http://localhost:8000/bootstrap/pets - Bootstrap pets demo
# http://localhost:8000/material/pets - Material Design pets demo
# http://localhost:8000/docs - Auto-generated Swagger UI
# http://localhost:8000/redoc - ReDoc documentation
```

## Architecture Notes

### Form Generation Approach
- **All forms are auto-generated** using pydantic-forms library
- `render_form_html()` generates complete form HTML from Pydantic models
- `ListLayout.render()` creates dynamic lists with add/remove functionality
- **No manual HTML form creation** - everything is generated from models

### ListLayout Implementation
- Proper layout type (not input type)
- Dynamic JavaScript for add/remove
- Framework-agnostic rendering
- Collapsible card support

### Code Organization
- Shared models for consistency
- Framework-specific demos
- Clean separation of concerns
- Reusable form validation logic
